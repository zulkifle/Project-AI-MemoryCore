## Project Structure

```
MyProject/
├── src/
│   ├── java/
│   │   ├── com/company/service/         ← SOAP provider + business logic
│   │   ├── com/company/ws/[client]/     ← wsimport-generated stubs (SOAP client)
│   │   ├── com/company/util/            ← utility classes
│   │   ├── META-INF/
│   │   │   ├── [ServiceName].xml        ← WSIT client config (auto-generated by NetBeans)
│   │   │   └── wsit-client.xml          ← WS client config
│   │   ├── config.properties            ← app config (DB, API keys)
│   │   └── log4j.properties             ← logging config
│   └── conf/
│       ├── [service].properties         ← WS endpoint URL, credentials
│       └── xml-resources/               ← referenced WSDLs
├── web/
│   └── WEB-INF/
│       ├── web.xml                      ← servlet mappings
│       ├── jboss-web.xml                ← WildFly context root
│       ├── jax-ws-catalog.xml           ← JAX-WS catalog
│       └── wsdl/                        ← local WSDL copies (one per client)
└── nbproject/
    ├── project.properties               ← build config: javac version, war name, classpath
    ├── project.xml                      ← NetBeans project definition
    ├── jax-ws.xml                       ← service + client definitions
    └── jaxws-build.xml                  ← Ant build extension for wsimport
```

## Key Config Files

### 1. `web/WEB-INF/jboss-web.xml` — WildFly Context Root

```xml
<?xml version="1.0" encoding="UTF-8"?>
<jboss-web version="8.0"
    xmlns="http://www.jboss.com/xml/ns/javaee"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://www.jboss.com/xml/ns/javaee
        http://www.jboss.org/schema/jbossas/jboss-web_8_0.xsd">
  <context-root>/[AppName]/[ServiceName]</context-root>
</jboss-web>
```

> **Example**: `/MyTrustPilotAPI/RoamingWS` → deployed at `http://host:8080/MyTrustPilotAPI/RoamingWS/`

### 2. `web/WEB-INF/web.xml` — Servlet Mappings

```xml
<?xml version="1.0" encoding="UTF-8"?>
<web-app version="3.1"
    xmlns="http://xmlns.jcp.org/xml/ns/javaee"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/javaee
        http://xmlns.jcp.org/xml/ns/javaee/web-app_3_1.xsd">

    <!-- Register plain HttpServlets here -->
    <servlet>
        <servlet-name>MyServlet</servlet-name>
        <servlet-class>com.company.service.MyServlet</servlet-class>
    </servlet>
    <servlet-mapping>
        <servlet-name>MyServlet</servlet-name>
        <url-pattern>/MyServlet</url-pattern>
    </servlet-mapping>

    <!-- NOTE: @WebService SOAP providers are NOT registered here.
         WildFly auto-publishes them based on @WebService annotation. -->

    <session-config>
        <session-timeout>30</session-timeout>
    </session-config>
</web-app>
```

### 3. `nbproject/jax-ws.xml` — Service + Client Definition

NetBeans reads this to know which classes to expose as SOAP services and which WSDLs to import.

```xml
<?xml version="1.0" encoding="UTF-8"?>
<jax-ws xmlns="http://www.netbeans.org/ns/jax-ws/1">

  <!-- SOAP Provider: exposes this class as a WS endpoint -->
  <services>
    <service name="[ServiceClassName]">
      <implementation-class>com.company.service.[ServiceClassName]</implementation-class>
    </service>
  </services>

  <!-- SOAP Client: imports WSDL and generates stubs via wsimport -->
  <clients>
    <client name="[ClientName]">
      <wsdl-url>[WSDL_URL]?wsdl</wsdl-url>
      <local-wsdl-file>[host]/[path]/[ServiceName].wsdl</local-wsdl-file>
      <package-name>com.company.ws.[clientpackage]</package-name>
      <catalog-file>catalog.xml</catalog-file>
      <wsimport-options>
        <wsimport-option>
          <wsimport-option-name>extension</wsimport-option-name>
          <wsimport-option-value>true</wsimport-option-value>
        </wsimport-option>
        <wsimport-option>
          <wsimport-option-name>verbose</wsimport-option-name>
          <wsimport-option-value>true</wsimport-option-value>
        </wsimport-option>
        <wsimport-option>
          <wsimport-option-name>wsdlLocation</wsimport-option-name>
          <wsimport-option-value>http://localhost:8080/[path]/[ServiceName]?wsdl</wsimport-option-value>
        </wsimport-option>
        <wsimport-option>
          <wsimport-option-name>xnocompile</wsimport-option-name>
          <wsimport-option-value>true</wsimport-option-value>
        </wsimport-option>
        <wsimport-option>
          <wsimport-option-name>xendorsed</wsimport-option-name>
          <wsimport-option-value>true</wsimport-option-value>
        </wsimport-option>
      </wsimport-options>
    </client>
  </clients>

</jax-ws>
```

> After saving this file in NetBeans: right-click project → **Clean and Build** to trigger wsimport and generate stubs into `build/generated-sources/jax-ws/`.

### 4. `nbproject/project.properties` — Build Config

Key lines to set for JDK 8 + WildFly:

```properties
j2ee.platform=1.7-web
j2ee.server.type=WildFly
javac.source=1.8
javac.target=1.8
war.name=[YourWAR].war

# External JARs bundled into WEB-INF/lib
file.reference.log4j.jar=path/to/log4j.jar
file.reference.json-simple.jar=path/to/json-simple-1.1.jar
file.reference.commons-codec.jar=path/to/commons-codec-1.9.jar

javac.classpath=\
    ${file.reference.log4j.jar}:\
    ${file.reference.json-simple.jar}:\
    ${file.reference.commons-codec.jar}
```

> WildFly's own JAX-WS classes are provided via `j2ee.platform.classpath` (auto-configured by NetBeans when server type = WildFly). Do NOT bundle `jaxws-rt.jar` — it conflicts with WildFly's built-in runtime.

## SOAP Provider — `@WebService` Pattern

A POJO annotated with `@WebService` is automatically published by WildFly as a SOAP endpoint.

```java
package com.company.service;

import javax.annotation.Resource;
import javax.jws.WebService;
import javax.jws.WebMethod;
import javax.jws.WebParam;
import javax.xml.ws.WebServiceContext;
import org.apache.log4j.Logger;

@WebService(serviceName = "MySignerService")
public class MySignerService {

    static Logger logger = Logger.getLogger(MySignerService.class.getName());

    @Resource
    WebServiceContext wsctx;  // inject WS context (for reading HTTP headers, etc.)

    @WebMethod(operationName = "SignData")
    public MyResponseData SignData(
            @WebParam(name = "UserID") String userID,
            @WebParam(name = "DataToSign") String dataToSign) {

        logger.info("[MySignerService] SignData starts — UserID=" + userID);

        MyServiceHelper helper = new MyServiceHelper();
        MyResponseData response = helper.doSign(userID, dataToSign, wsctx);

        logger.info("[MySignerService] SignData done — ErrCode=" + response.getErrCode());
        return response;
    }
}
```

**Rules:**
- `serviceName` in `@WebService` = the WSDL service name
- `operationName` in `@WebMethod` = WSDL operation name
- `name` in `@WebParam` = WSDL parameter element name — must match exactly
- WildFly publishes this at: `http://host:8080/[context-root]/MySignerService?wsdl`
- This class is NOT a Servlet — do NOT register it in `web.xml`
- Also register in `nbproject/jax-ws.xml` under `<services>` so NetBeans tracks it

## SOAP Client — wsimport Stubs Pattern

After wsimport generates stubs from a WSDL, use them like this:

```java
package com.company.service;

import com.company.ws.roaming.RoamingService;
import com.company.ws.roaming.RoamingService_Service;
import com.company.ws.roaming.PdfHashToSignData;
import com.company.ws.roaming.PdfHashSignedDataResponse;
import javax.xml.ws.BindingProvider;
import java.util.Map;
import org.apache.log4j.Logger;

public class RoamingSignerHelper {

    static Logger logger = Logger.getLogger(RoamingSignerHelper.class.getName());

    private String wsUrl;
    private String wsUser;
    private String wsPass;

    public RoamingSignerHelper() {
        // load from properties
        ReadConfig cfg = new ReadConfig();
        wsUrl  = cfg.getProperty("roaming.ws.url");
        wsUser = cfg.getProperty("roaming.ws.user");
        wsPass = cfg.getProperty("roaming.ws.pass");
    }

    public PdfHashSignedDataResponse callSignHash(String userID, String pin, ...) {
        try {
            // 1. Create service from generated stubs
            RoamingService_Service svc = new RoamingService_Service();
            RoamingService port = svc.getRoamingServicePort();

            // 2. Override endpoint URL
            Map<String, Object> ctx = ((BindingProvider) port).getRequestContext();
            ctx.put(BindingProvider.ENDPOINT_ADDRESS_PROPERTY, wsUrl);

            // 3. Set HTTP headers if required by the WS
            // ctx.put(MessageContext.HTTP_REQUEST_HEADERS, Map.of(
            //     "Username", List.of(wsUser),
            //     "Password", List.of(wsPass)
            // ));

            // 4. Call the operation
            PdfHashSignedDataResponse result = port.signPdfHashBulk(userID, pin, ...);

            logger.info("[callSignHash] result ErrCode=" + result.getErrCode());
            return result;

        } catch (Exception e) {
            logger.error("[callSignHash] Exception: " + e.getMessage(), e);
            return null;
        }
    }
}
```

**Rules:**
- Generated stub classes live in `com.company.ws.[clientpackage]` (from `jax-ws.xml`)
- `[ServiceName]_Service` is the entry point; `get[ServiceName]Port()` gets the callable port
- Always override `ENDPOINT_ADDRESS_PROPERTY` — never hardcode URL into the stub
- Check result for null before using it

## Properties File Pattern

Store WS endpoint, credentials, and app config in `src/conf/[service].properties`:

```properties
# Endpoint URLs (comment out non-active environment)
#ws.url=https://staging.example.com/WSAPI/Service?wsdl
ws.url=https://prod.example.com/WSAPI/Service?wsdl

ws.user=myuser
ws.pass=MyEncryptedPass

ws.stage=prod

# AES encryption key+IV (for encrypting PIN before sending)
app.encryption.key=jC9L4V9kvtJXSbwu
app.encryption.iv=vK5jdztVXAZk8Gxh
```

> Properties file is loaded at runtime from the classpath (`src/conf/` is on the classpath via Ant).

## Reading Properties

```java
public class ReadConfig {
    private Properties props = new Properties();

    public ReadConfig() {
        try (InputStream is = getClass().getClassLoader()
                .getResourceAsStream("roaming.properties")) {
            props.load(is);
        } catch (Exception e) {
            // log error
        }
    }

    public String getProperty(String key) {
        return props.getProperty(key);
    }
}
```

## Build & Deploy

### Build .war in NetBeans
1. Right-click project → **Clean and Build**
2. WAR produced at: `dist/[YourWAR].war`

### Deploy to WildFly
- **Hot deploy**: copy WAR to `$WILDFLY_HOME/standalone/deployments/`
- **Admin console**: `http://host:9990` → Deployments → Upload
- WildFly auto-generates `.deployed` marker when deployment succeeds; `.failed` on error

### Verify WSDL published
After deployment: `http://host:8080/[context-root]/[ServiceName]?wsdl`

## Common Issues & Fixes

| Error | Cause | Fix |
|---|---|---|
| WSDL not found at `?wsdl` | Context root mismatch | Check `jboss-web.xml` `<context-root>` matches URL you're hitting |
| `ClassNotFoundException` on WS class | Not in WAR's WEB-INF/lib or classpath | Verify library jars are referenced in `project.properties` under `javac.classpath` |
| wsimport fails: "Cannot read WSDL" | WSDL URL unreachable at build time | Use `<local-wsdl-file>` in `jax-ws.xml` instead; save WSDL under `web/WEB-INF/wsdl/` |
| `ProviderImpl not found` | Accidentally bundled JAX-WS jar conflicts with WildFly | Remove `jaxws-rt.jar` from `WEB-INF/lib`; WildFly provides JAX-WS |
| WS returns old code after redeploy | WildFly caching | Touch `.war` file or undeploy/redeploy manually via admin console |
| `@WebParam name` mismatch | Java param name ≠ WSDL element name | Set `@WebParam(name = "ExactWSDLName")` on each parameter |

## New Project Checklist

When Dejul says "new WildFly SOAP WS project":

1. **Ask for project info** (service name, WAR name, context root, WS to consume if any)
2. **Create folder structure** as shown above
3. **Create `jboss-web.xml`** with correct context root
4. **Create `web.xml`** with servlet mappings (only for HttpServlets, not SOAP providers)
5. **Create `jax-ws.xml`** with service + client entries
6. **Create service class** with `@WebService` + `@WebMethod` skeleton
7. **Create `.properties` file** in `src/conf/`
8. **Set `project.properties`** — javac.source=1.8, jar references, war name
9. **Run wsimport** (Clean and Build) to generate client stubs if consuming a WSDL
